# Triage Prompt — ранжирование всех проектов перед аудитом

Запускается **один раз** перед серией аудитов. Результат — таблица приоритетов, по ней решаем порядок.

---

## Промпт

```
# Triage задача: все активные проекты

Прочитай:
1. /home/igor/CLAUDE.md — список App UUIDs (таблица проектов)
2. ~/vault/01-Projects/BOARD.md — статусы и приоритеты
3. ~/vault/01-Projects/*/README.md — бизнес-контекст каждого

## Режим: TRIAGE ONLY
Не открывай код проектов. Только метаданные + README + статус.

## Задача

Создай таблицу в `~/vault/01-Projects/Audit-Campaign/triage-<YYYY-MM-DD>.md`:

| Проект | Бизнес-критичность | Прод-юзеры | Деньги/PII | Сложность кода | Audit-приоритет |
|---|---|---|---|---|---|

Колонки:
- **Бизнес-критичность:** P0 (без него деньги/репутация теряются) / P1 (важный inhouse-tool) / P2 (полезный) / P3 (хобби/отключён)
- **Прод-юзеры:** да / нет (только Игорь)
- **Деньги/PII:** да (платежи, личные данные, авторизация) / нет
- **Сложность кода:** S (<500 строк) / M (500-3k) / L (>3k) — грубая оценка из размера репо
- **Audit-приоритет:** 1 (немедленно) / 2 (в этом квартале) / 3 (можно отложить) / SKIP (rss-bot, отключённые)

## Правила ранжирования

1. **Audit-приоритет 1** = есть прод-юзеры + (деньги ИЛИ PII)
2. **Audit-приоритет 2** = есть прод-юзеры без денег/PII, ИЛИ inhouse-tool на котором держится инфра
3. **Audit-приоритет 3** = только Игорь юзер, сломается — не страшно
4. **SKIP** = rss-bot (отключён), любой проект в архиве

## После таблицы

Дай рекомендацию:
"Начать с: <PROJECT_NAME>. Причина: <одна фраза>. Команда: открой новую сессию с промптом из ~/vault/03-Resources/_templates/audit-prompt.md, замени <PROJECT> на <PROJECT_NAME>."
```
